How to Create a Plugin for ConvertKit

This tutorial guides you through creating a custom plugin for ConvertKit.

Step 1: Understand the Plugin Structure

Plugins are Python modules that define a conversion function. Each plugin must be placed in the plugins/ directory and follow this structure:

  • Input Format: The file extension the plugin accepts (e.g., 'jpg', 'pdf').
  • Output Format: The file extension the plugin produces (e.g., 'bmp', 'docx').
  • Convert Function: A function named convert(file_path) that takes the input file path and returns the output file path.

Step 2: Create Your Plugin Module

Create a new Python file in the plugins/ directory, e.g., my_plugin.py. Here's a template: 


# plugins/my_plugin.py
from PIL import Image

input_format = "jpg"
output_format = "bmp"

def convert(file_path):
    img = Image.open(file_path)
    new_path = file_path.replace(".jpg", "_converted.bmp")
    img.save(new_path, "BMP")
    return new_path

Step 3: Test Your Plugin Locally

Run the Flask app and test your plugin by uploading a file of the input type. Ensure the conversion works as expected.

Step 4: Contribute via Pull Request

To add your plugin to the official repository:

  1. Fork the repository at https://github.com/sebcun/ConvertKit.
  2. Clone your fork locally: git clone https://github.com/yourusername/ConvertKit.git
  3. Create a new branch: git checkout -b add-my-plugin
  4. Add your plugin file to the plugins/ directory.
  5. Commit your changes: git add . && git commit -m "Add my_plugin for jpg to bmp conversion"
  6. Push to your fork: git push origin add-my-plugin
  7. Open a pull request on GitHub, describing your plugin and its functionality.

Additional Notes

  • Check existing plugins in the plugins/ directory for examples.
